# Discovery Notes — 学童保育Webアプリ（がくどう）

## 1. ターゲットユーザー

このアプリには3種類のユーザーがいる：

1. **一般訪問者（未ログイン）** — 学童保育のHPとして、施設の紹介や基本情報を閲覧する人。入会を検討している保護者や、説明会の日程を知りたい新一年生の親など。デバイスはスマホ・PC問わず。

2. **保護者（ログインユーザー）** — 学童に通う子どもの親。スマホで手軽に子どもの写真・動画を確認したり、先生からの連絡事項を読む。30人の生徒に対し、各家庭1〜2アカウント程度。主にスマホからの利用を想定。

3. **先生（管理者ユーザー）** — 6名のスタッフ。写真・動画のアップロード、連絡事項の登録、生徒のQRコード入退場記録の管理を行う。スマホまたはPC・タブレットから利用。

現在は1施設・生徒30名の規模だが、将来的には複数施設への展開を予定している。

## 2. 課題・ペイン

- **情報がLINEに散在** — 写真も連絡事項もLINEで流れるため、過去の情報を探しにくい。学童専用の情報共有基盤がない。
- **入退場管理の固定費** — KAZASUを導入しているが、多機能すぎて使いこなせておらず、入退場記録だけのために高い固定費を払っている。
- **入退場記録の活用** — 延長保育の追加料金計算に入退場時間を使いたいが、現状では効率的にできていない。
- **HPが機能していない** — 既存HPはしばらく更新されておらず見栄えも悪い。新規の保護者から説明会の案内がわからないという声があった。

## 3. 現在の代替手段

| 業務 | 現在の方法 | 課題 |
|------|-----------|------|
| 写真・動画共有 | LINE | 流れてしまう、整理しにくい |
| 連絡事項 | LINE | 同上 |
| 入退場記録 | KAZASU | 固定費が高い、機能を持て余している |
| HP | 既存サイト（放置状態） | 更新されていない、見栄えが悪い、説明会情報が見つからない |

## 4. 成功指標

- KAZASUを解約できる — 入退場記録機能が代替として稼働し、固定費を削減
- 先生の業務負担が軽減 — 写真共有・連絡事項の配信がアプリで完結し、LINEでの個別対応が減る
- 親の満足度向上 — 子どもの様子をスマホで手軽に確認でき、連絡事項の見落としが減る
- HP経由の情報発信が機能する — 説明会の案内などが一般公開ページで確認でき、新規入会希望者が迷わない

## 5. スコープ

### v1に入れるもの
- 公開HP（施設紹介・説明会案内・お知らせ）
- ログイン機能（保護者 / 先生の2ロール）
- 写真・動画の共有（先生がアップロード → 保護者が閲覧）
- 連絡事項の配信（先生が登録 → 保護者に表示）
- QRコード入退場記録（タブレット端末でカードをかざす）
- 入退場履歴の閲覧（先生：全員分、保護者：自分の子どもだけ）

### v1から明示的に除外
- 延長保育料金の自動計算・請求
- 複数施設対応
- 保護者同士のコミュニケーション機能
- プッシュ通知

## 6. 制約条件

- **技術レベル**: エンジニア（React + TypeScript、SpringBoot + Kotlin の実務経験あり）
- **フレームワーク**: フロントはReact + TypeScript に慣れている
- **予算**: できる限り無料枠で運用したい（ホスティング・DB・ストレージ含め）
- **コンプライアンス**: 子どもの写真・動画を扱うため、ログイン必須でアクセス制限が必要（一般公開はNG）
