# Discovery Notes — 学童保育Webアプリ（gakudo）

## 1. ターゲットユーザー

**保護者（メインユーザー）：** 共働きで忙しい母親が典型。スマホを主デバイスとして、通勤中や昼休みなどの隙間時間に利用する。子供が学童に無事到着したかをリアルタイムで確認したい。連絡事項やイベント写真・動画を手軽にチェックしたい。プッシュ通知で見逃しを防ぎたい。

**先生（指導員）：** 児童の帰宅後にPCまたはスマホから連絡事項をまとめて配信する。スマホで撮影した写真を気軽にアップロードして共有。施設のPCに接続したウェブカメラでQRコードを読み取り、児童の入退場を自動記録する。入退場データから延長保育料金を自動計算し、手作業を削減したい。

**管理者：** 児童・保護者・先生のアカウント管理、QRコード発行、延長保育料金のルール設定を行う。（先生が兼任する可能性あり）

**利用形態：** B2B（学童保育施設向けWebアプリ）。現在QR入退場管理に有料サービスを利用しており、自前化によるコスト削減が主要な動機。

## 2. 問題と課題

**中心の課題：** 父母が運営に深く関与する学童保育において、連絡・情報共有・入退場管理・請求が複数のツール（LINE、有料入退場サービス、紙）に分散しており、慢性的な非効率とコストが発生している。痛みの度合いは7/10程度で、急性ではないが長年の蓄積した不満がある。

**具体的な痛み：**
- **連絡手段がLINE頼み：** グループLINEで連絡事項や資料（PDF）を共有しているが、過去の資料が流れて見つけにくい。PCで作成した資料をスマホに転送してからLINEに投稿する手間がある。
- **LINEグループの乱立と管理負荷：** 役員会、係別など複数のLINEグループが必要。年1回の役員交代時にグループの再編成が面倒。
- **入退場管理の過剰なコスト：** 有料サービスを利用中で機能自体は問題ないが、入退場管理だけでよいのに不要な機能分のコストを払っている。
- **延長保育料金の手計算・紙運用：** 基本料金は学年で一律だが、延長保育の利用頻度は家庭ごとに異なる。個別計算した請求を紙で手渡ししており、計算・配布の手間がかかる。

**発生頻度：** 連絡事項は日常的（ほぼ毎日）、資料共有は月1回の役員会前後、請求は月次、役員入替は年1回。

## 3. 現在の代替手段

**連絡・情報共有：LINE（無料）**
- 保護者・先生間の日常連絡、資料共有に利用。無料で手軽だが、資料が流れる・グループ管理が煩雑・PCからの資料転送が面倒という課題あり。

**入退場管理：KAZASU（有料 約100円/児童/月）**
- QRコードによる入退場管理を提供。機能自体は問題なく動作。ただし不要な機能が含まれ、30人×100円＝約3,000円/月のコストに不満。導入経緯を知る役員は既に退任。

**延長保育料金の計算・請求：手作業（紙）**
- 入退場記録をもとに先生が手計算し、紙で各家庭に個別配布。

**HP（外部向け情報発信）：既存HPはあるが放置状態**
- 学童のHPは存在するがメンテナンスされていない。入園説明会の日程など、入園希望者向けの情報が外部から見つけにくい状態。今年入園した保護者から「情報が見つからず手間取った」との声あり。外部向けの情報発信動線が整備されていない。

**写真・動画共有：なし（LINEで断片的に共有する程度）**

**乗り換えの条件：** 入退場管理＋連絡＋請求＋外部向けHP がひとつのアプリに統合され、KAZASU＋LINE＋紙運用＋放置HPの4つを置き換えられること。

## 4. 成功指標

**成功の定義（3ヶ月後の目標）：**
- KAZASUを解約し、入退場管理を自前アプリに完全移行（月約3,000円のコスト削減）
- 先生の事務作業時間が削減され、児童と過ごす時間が増加
- 保護者がLINEではなくアプリで連絡事項・資料を確認する習慣が定着
- 延長保育の請求が自動化され、紙の手渡しが廃止
- 入園希望者がHPから説明会情報に容易にアクセスできる

**理想的な追加成果：**
- 先生の業務効率化により人件費の削減にも貢献
- 保護者から「子供をより手厚く見てもらえている」と実感される

**失敗の定義：**
- 先生や保護者にとって逆に手間が増える（LINEより面倒）
- 入退場管理の不具合により児童の安全管理に支障が出る（最も深刻）
- 一部の保護者から「前の方がよかった」と苦情が出る

**最重要リスク：** 入退場管理は児童の安全に直結するため、信頼性・正確性が最優先。移行期間中はKAZASUとの並行運用を検討すべき。

## 5. v1スコープ

### v1に含める
1. **学童HP（公開ページ）** — 施設情報、説明会日程、アクセスなどの外部向け情報発信
2. **アカウント管理** — 保護者・先生・管理者の3ロールでのログイン
3. **児童・保護者の紐付け管理** — 児童登録、保護者との関連付け
4. **連絡事項の配信＋プッシュ通知** — 先生から保護者への一斉配信、既読管理
5. **資料共有（掲示板型）** — PDF等をカテゴリ別に整理、過去分も検索可能
6. **グループ管理** — 役員会・係別などの情報共有グループ
7. **年度更新・役員引継ぎ** — メンバー入替を簡単に実行できる仕組み
8. **写真・動画の共有** — 先生がスマホから気軽にアップロード、保護者が閲覧
9. **QRコード発行** — 児童ごとにユニークなQRコードを生成・印刷
10. **QRコード入退場管理** — PCのウェブカメラで読み取り、自動記録
11. **入退場履歴の閲覧** — 保護者がリアルタイムで子供の到着・退出を確認
12. **延長保育料金の自動計算** — 入退場データから自動算出
13. **請求書のオンライン確認** — 保護者がアプリ上で月次請求を確認（紙廃止）

### v1ではやらない（明示的にスコープ外）
- オンライン決済（クレカ・口座振替）
- チャット機能（双方向メッセージ）
- 出席簿・学習記録管理
- 複数施設の統合管理（マルチテナント）
- スマホネイティブアプリ（PWA/Webで対応）
- 多言語対応

## 6. 制約条件

- **ユーザー技術レベル：** プロのエンジニア（React得意）
- **フレームワーク希望：** React（フロントエンド）
- **インフラ経験：** 業務でAWS利用経験あり
- **ホスティング希望：** なるべく安く、できれば無料枠で運用したい（Vercel + Supabase等のフリーティアが候補）
- **予算：** 最小限。現在KAZASUに月約3,000円支払っているため、それ以下で運用できることが理想
- **タイムライン：** 特に制約なし
- **コンプライアンス・データ管理：**
  - 特定の法規制要件は現時点ではなし
  - **写真・動画のプライバシーが最重要：** 児童の写真は保護者と先生のみ閲覧可能（認証必須）
  - 一部の写真は保護者の許可を得た上でHP（公開ページ）に掲載可能
  - 公開/非公開の区別を写真単位で管理できる仕組みが必要
