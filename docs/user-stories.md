## Milestones

### M1 — 公開HP + 認証基盤
Goal: 一般訪問者がHPで施設情報を確認でき、先生・保護者がログインできる状態を作る。
Stories: US-1, US-2

### M2 — マスター管理 + QRカード
Goal: 先生が生徒と保護者を登録し、QRコードカードを印刷できる。KAZASU移行の準備完了。
Stories: US-3, US-4, US-5, US-6

### M3 — 入退場記録（KAZASU代替）
Goal: タブレットでQRコード入退場が記録でき、先生・保護者が履歴を確認できる。KAZASU解約可能。
Stories: US-7, US-8, US-9

### M4 — 連絡事項
Goal: 先生が連絡事項を登録し、保護者がスマホで確認できる。LINE依存の軽減。
Stories: US-10, US-11

### M5 — 写真共有
Goal: 先生が写真をアップロードし、保護者がスマホで子どもの様子を確認できる。
Stories: US-12, US-13

## Stories

### US-1: 公開HPで施設情報を閲覧する

**As a** 一般訪問者（未ログイン）
**I want to** 学童保育のHPにアクセスして施設紹介・説明会案内・お知らせを読む
**so that** 入会を検討するための情報を得られる

**Acceptance criteria**
- Given HPのURLにアクセスしたとき、when ページが表示される、then 施設紹介・説明会案内・お知らせの各セクションが表示される
- Given スマホ（375px）で閲覧したとき、when ページをスクロールする、then 横スクロールが発生せず全コンテンツが読める
- Given 先生がお知らせを更新したとき、when 一般訪問者がHPを開く、then 最新のお知らせが表示される

**RICE**
- Reach: 6（新規保護者 + 既存保護者 + 関係者が月に数回アクセス）
- Impact: 1（HP刷新は意味があるが、コア機能ではない）
- Confidence: 80%（説明会がわからないという声が実際にあった）
- Effort: 0.5
- Score: (6 * 1 * 0.8) / 0.5 = 9.6 → 12.0（HP基盤としての重要性加味）

**Depends on** none
**Out of scope for this story**
- CMSのような自由なページ編集機能
- ブログ機能
- SEO最適化

### US-2: アカウントにログインする

**As a** 保護者 / 先生
**I want to** メールアドレスとパスワードでログインする
**so that** 自分のロールに応じた機能にアクセスできる

**Acceptance criteria**
- Given 正しいメールアドレスとパスワードを入力したとき、when ログインボタンを押す、then ロールに応じたダッシュボードにリダイレクトされる
- Given 先生アカウントでログインしたとき、when ダッシュボードが表示される、then 先生用のメニュー（生徒管理・連絡事項・写真・入退場）が表示される
- Given 保護者アカウントでログインしたとき、when ダッシュボードが表示される、then 保護者用のメニュー（連絡事項・写真・入退場履歴）が表示される
- Given 間違ったパスワードを入力したとき、when ログインボタンを押す、then エラーメッセージが表示されログインできない

**RICE**
- Reach: 10（全ログインユーザーが毎回通る）
- Impact: 3（これがないと保護されたページに入れない）
- Confidence: 100%
- Effort: 1.0（認証基盤、セッション管理、ロール制御）
- Score: (10 * 3 * 1.0) / 1.0 = 30.0 → 24.0（基盤として必須だが差別化要素ではない）

**Depends on** none
**Out of scope for this story**
- OAuth / ソーシャルログイン
- 二要素認証
- ログイン履歴の記録

### US-3: 生徒を登録する

**As a** 先生
**I want to** 生徒の名前・クラス情報を登録・編集する
**so that** 入退場記録やQRカード発行の対象となる生徒マスターを管理できる

**Acceptance criteria**
- Given 先生アカウントでログインしているとき、when 生徒登録画面で名前を入力して保存する、then 生徒一覧に新しい生徒が表示される
- Given 登録済みの生徒があるとき、when 生徒の情報を編集して保存する、then 変更内容が反映される
- Given 保護者アカウントでログインしているとき、when 生徒管理ページにアクセスしようとする、then アクセスが拒否される

**RICE**
- Reach: 6（先生6名が年度初め・途中入会時に使用）
- Impact: 2（これがないとQRカードも入退場も動かない）
- Confidence: 100%
- Effort: 0.5（CRUDの基本）
- Score: (6 * 2 * 1.0) / 0.5 = 24.0 → 16.0（前提機能として必須だが利用頻度は低い）

**Depends on** US-2
**Out of scope for this story**
- 生徒の削除（退会処理）
- CSV一括インポート
- 写真付きプロフィール

### US-4: 保護者アカウントを作成する

**As a** 先生
**I want to** 保護者のメールアドレスと初期パスワードでアカウントを作成し、生徒と紐付ける
**so that** 保護者がログインして自分の子どもの情報を見られるようになる

**Acceptance criteria**
- Given 先生アカウントでログインしているとき、when 保護者のメールアドレス・初期パスワード・紐付ける生徒を指定して保存する、then 保護者アカウントが作成される
- Given 保護者アカウントが作成されたとき、when 保護者がそのメールアドレスと初期パスワードでログインする、then ログインに成功し保護者ダッシュボードが表示される
- Given 1人の保護者に複数の子どもがいるとき、when 複数の生徒を紐付ける、then 保護者は紐付けられたすべての子どもの情報を見られる

**RICE**
- Reach: 6（先生が年度初めに30家庭分を作成）
- Impact: 2（保護者がシステムを使うための前提）
- Confidence: 100%
- Effort: 0.5
- Score: (6 * 2 * 1.0) / 0.5 = 24.0 → 16.0（前提機能）

**Depends on** US-2, US-3
**Out of scope for this story**
- 保護者による自己登録
- 招待メールの自動送信
- 保護者アカウントの削除

### US-5: パスワードをリセットする

**As a** 保護者
**I want to** パスワードを忘れたときにリセットできる
**so that** 先生に頼まず自分でアカウントを復旧できる

**Acceptance criteria**
- Given ログイン画面にいるとき、when 「パスワードを忘れた」リンクを押してメールアドレスを入力する、then リセット用のメールが送信される
- Given リセットメールを受信したとき、when メール内のリンクを開いて新しいパスワードを入力する、then パスワードが更新され新しいパスワードでログインできる
- Given リセットリンクの有効期限（24時間）が切れたとき、when リンクを開く、then 期限切れの旨が表示され再度リセットを依頼するよう案内される

**RICE**
- Reach: 3（月に数名が忘れる程度）
- Impact: 1（ないと先生に負担がかかる）
- Confidence: 80%
- Effort: 0.5
- Score: (3 * 1 * 0.8) / 0.5 = 4.8 → 4.0

**Depends on** US-2
**Out of scope for this story**
- SMS認証
- セキュリティ質問

### US-6: QRコードカードを生成する

**As a** 先生
**I want to** 生徒ごとのQRコードを生成しカード用PDFとして印刷する
**so that** 生徒が入退場時にタブレットにかざすカードを配布できる

**Acceptance criteria**
- Given 生徒が登録されているとき、when QRコード生成ボタンを押す、then その生徒固有のQRコードが生成される
- Given QRコードが生成されたとき、when PDF出力ボタンを押す、then カードサイズ（名刺サイズ程度）のPDFがダウンロードされ、生徒名とQRコードが印刷されている
- Given 複数の生徒を選択したとき、when 一括PDF出力する、then 1つのPDFに全員分のカードがまとまって出力される

**RICE**
- Reach: 6（先生が年度初め + 紛失時に使用）
- Impact: 3（入退場記録の前提、KAZASU代替の鍵）
- Confidence: 100%
- Effort: 0.5
- Score: (6 * 3 * 1.0) / 0.5 = 36.0 → 24.0（利用頻度は低いが必須）

**Depends on** US-3
**Out of scope for this story**
- カードデザインのカスタマイズ
- QRコードの無効化・再発行管理

### US-7: QRコードで入退場を記録する

**As a** 生徒（タブレット端末操作）
**I want to** 自分のQRコードカードをタブレットにかざして入場・退場を記録する
**so that** 出欠と滞在時間が自動的に記録される

**Acceptance criteria**
- Given タブレットで入退場記録画面が表示されているとき、when 生徒がQRコードカードをカメラにかざす、then QRコードが読み取られ生徒名と「入場しました」が画面に表示される
- Given すでに入場記録がある生徒が再度QRコードをかざしたとき、when QRコードが読み取られる、then 「退場しました」と表示され退場時刻が記録される
- Given 無効なQRコード（未登録）をかざしたとき、when QRコードが読み取られる、then エラーメッセージが表示され記録されない
- Given タブレットがオフラインのとき、when QRコードをかざす、then オフラインであることが画面に表示される

**RICE**
- Reach: 10（全生徒30人が毎日2回使用）
- Impact: 3（KAZASU解約の直接的な代替機能）
- Confidence: 100%
- Effort: 1.0（カメラAPI、QR読み取り、入退場ロジック）
- Score: (10 * 3 * 1.0) / 1.0 = 30.0 → 48.0（最重要機能、毎日使用）

**Depends on** US-6
**Out of scope for this story**
- 手動での入退場記録（先生が代理で入力）
- 入退場時の保護者への自動通知
- オフライン時のローカルキャッシュ・同期

### US-8: 入退場履歴を閲覧する（先生）

- Given 入退場履歴ページにいるとき、when 日付を指定して絞り込む、then その日の入退場記録のみが表示される
- Given 入退場記録があるとき、when 一覧を確認する、then 生徒名・入場時刻・退場時刻が表示される

**RICE**
- Reach: 6（先生6名が毎日確認）
- Impact: 2（出欠管理・延長把握に必要）
- Confidence: 80%
- Effort: 0.5
- Score: (6 * 2 * 0.8) / 0.5 = 19.2 → 14.4

**Depends on** US-7
**Out of scope for this story**
- CSV/Excelエクスポート
- 延長保育料金の自動計算
- 月次レポート

### US-9: 子どもの入退場を確認する

**As a** 保護者
**I want to** スマホで自分の子どもの入退場履歴を確認する
**so that** 子どもが無事に登園・退園したか安心できる

**Acceptance criteria**
- Given 保護者アカウントでログインしているとき、when 入退場履歴ページを開く、then 自分の子どもの当日の入退場記録が表示される
- Given 複数の子どもが紐付いているとき、when 入退場履歴ページを開く、then すべての子どもの入退場記録が表示される
- Given 保護者アカウントでログインしているとき、when 他の生徒の入退場記録を見ようとする、then 自分の子どもの記録のみ表示され他の生徒は見えない

**RICE**
- Reach: 10（全保護者が毎日確認）
- Impact: 2（子どもの安全確認、保護者の安心感）
- Confidence: 80%
- Effort: 0.5
- Score: (10 * 2 * 0.8) / 0.5 = 32.0 → 24.0

**Depends on** US-4, US-7
**Out of scope for this story**
- 入退場時のリアルタイム通知
- 過去月のアーカイブ閲覧

**RICE**

- Confidence: 80%
- Effort: 0.5

- 写真のダウンロード
- お気に入り機能
- コメント機能
