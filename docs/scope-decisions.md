# Scope decisions — 学童保育Webアプリ（がくどう）

## Recommendation

v1は「入退場記録 + 連絡事項 + 写真共有 + 公開HP」の4本柱で進める。動画共有はv1.1に回す。

最大のカットは動画共有。動画のアップロード・ストレージ・配信は技術的に重く、無料枠での運用を大きく圧迫する（動画1本で数十〜数百MB）。写真は1枚数MB以下で済むため無料枠でも現実的に運用可能。

最も早く「これは使える」と判断できるのは、先生がタブレットでQRコードをかざして入退場を記録し、保護者がスマホで子どもの登園を確認できた瞬間。ここまでを最短で届け、KAZASU解約を実現する。

## Lens findings

### Expand
- **生徒・保護者のマスター管理**: 先生が生徒を登録し、保護者アカウントと紐付ける機能が暗黙の前提。QRカード発行も入退場閲覧もこれがないと動かない。v1に追加。
- **QRコードカードの生成・印刷**: 生徒登録時にQRコードを生成しPDF出力する機能が必要。タブレットでかざすカードの元データ。v1に追加。

### Selectively expand
- **ログイン・アカウント管理**: 保護者の初回アカウント作成は先生が初期パスワード付きで作成し保護者に渡す運用とする。パスワードリセット機能は必要。OAuthやソーシャルログインはv1では不要。
- **連絡事項**: v1は「先生が登録、保護者が閲覧」のみ。既読管理はv2に回す。

### Preserve
- **公開HP（施設紹介・説明会案内）**: 実装コストが低く、新規保護者の課題を直接解決する。正しいスコープ。
- **入退場記録のタブレット＋QRカード方式**: KAZASUの代替として十分シンプルで、子どもが自分で操作できる。
- **v1除外項目（料金計算・複数施設・プッシュ通知）**: すべて正しい判断。

### Reduce
- **動画共有 → v1.1に回す**: ストレージ・配信コストが無料枠と相性が悪い。LINEで当面代替可能。KAZASU解約の学びには影響しない。

## In scope for v1 (revised)
- 公開HP（施設紹介・説明会案内・お知らせ）
- ログイン機能（保護者 / 先生の2ロール）
- 保護者アカウントの作成（先生が作成、初期パスワード付与）
- パスワードリセット
- 生徒マスター管理（先生が登録・編集）
- 生徒と保護者アカウントの紐付け
- QRコード生成・カード用PDF出力
- QRコード入退場記録（タブレット端末でカードをかざす）
- 入退場履歴の閲覧（先生：全員分、保護者：自分の子どもだけ）
- 写真の共有（先生がアップロード → 保護者が閲覧）
- 連絡事項の配信（先生が登録 → 保護者に表示）

## Deferred to v2 or later
- 動画の共有（ストレージ・配信コスト問題の解決後）
- 延長保育料金の自動計算・請求
- 複数施設対応
- 保護者同士のコミュニケーション機能
- プッシュ通知
- 連絡事項の既読管理
- OAuth / ソーシャルログイン

## Rationale for cuts (one-liner per cut)
- 動画共有: 無料枠でのストレージ・配信が非現実的、LINEで当面代替可能
- 料金計算: 入退場時間の記録さえあれば手動計算で対応可能
- 複数施設: 現在1施設のみ、需要が顕在化してから対応
- コミュニケーション機能: LINEが十分に機能している
- プッシュ通知: ブラウザでの確認で十分、技術的複雑性が高い
- 既読管理: 表示するだけで保護者の情報見落としは十分改善される
- OAuth: 先生が初期アカウントを作成する運用で十分
